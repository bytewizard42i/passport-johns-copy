# did:ray (RAY Network)

| Property | Value |
|----------|-------|
| Specification | [GitHub](https://github.com/pcrui/did-spec/blob/main/README.md) |
| Organization | Ray Peng |
| DID Format | `did:ray:<24-char-hash>` |

## Overview
did:ray is a DID method for the RAY network, providing blockchain-based digital identity services. The 24-character identifier is generated by hashing the public key. DDO (DID Description Object) documents are stored on-chain and publicly readable.

## Evaluation

### 1. Feasibility/Complexity
**Not self-contained.** The method requires RAY network blockchain:

- All DDO documents stored on-chain
- Smart contract (BIdContract) manages updates
- Uses DPOS and PBFT consensus algorithms
- Supports RSA, ECC, and SM2 cryptographic algorithms
- Resolution via HTTP query to `did.pengray.cn`

Zero-knowledge proofs available for enhanced privacy.

### 2. Ecosystem
**Very small.** Appears to be a regional (Chinese) project. Limited documentation and community activity.

### 3. Stability
**Early stage.** Limited production deployment evidence. Documentation primarily in Chinese.

## Recommendation
**No-go**

Requires RAY network blockchain infrastructure for all operations. HTTP resolution endpoint dependency adds another external requirement.
